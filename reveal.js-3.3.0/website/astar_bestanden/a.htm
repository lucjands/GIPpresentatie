                            amitp Mod Andreas a month ago

                            Since Dijkstra's visiting the nodes in order of distance, you can stop as soon as the head of the priority queue exceeds your range limit. Everything in the frontier (priority queue) has to be at least that far away, and every unvisited node has to be at least as far as anything in the frontier.

                                                Does the put method in Dijkstra algorithm check for the duplicate? I mean will it replace the value for an existing item or add another item all together?

                                                    amitp Mod Swaroop a month ago

                                                    It will add another item. Some people don't like this. I've written more about this topic here. Either you can try to replace the existing item, which makes your priority queue more complicated (often slower), or you can have another item, which makes your priority queue bigger (usually slower). Some priority queue data structures do support updating in place, but I am using a binary heap, which does not. Either way is fine! I chose to have the additional item, because it keeps the code smaller and simpler.

                                                                                              amitp Mod DSchouten 5 months ago

                                                                                              A* works with any graph structure, so you can represent platform jump and landing points as graph nodes, and the jumps and falls as graph edges. Annotate the graph edges with the type of move, whether it's a run/walk or a jump or a fall. That way, when A* returns a path P--Q--R--S you can see that the P--Q move is running right, Q--R is jumping up, R--S is moving right, etc. You might also take a look at this question on stackoverflow.
